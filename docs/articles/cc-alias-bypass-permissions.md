# Как я перестал кликать на каждое действие Claude Code и начал делегировать реально

## 88 раз в день. Каждый раз — пауза.

Представь: ты сидишь в потоке, прокручиваешь задачу в голове, и тут — **бам!** — всплывающее окно Claude Code.

> "Разрешить выполнение `npm install`?"

Клик. Пауза. Разрыв контекста.

За день набегает 80-100 таких остановок. Каждая — 2-3 секунды потерянного контекста. Каждая — микро-фрустрация, которая незаметно эродирует фокус.

Это я называю **approve loop**: задача → план → шаг → approve → шаг → approve → ... → ещё approve → ещё approve → стек.

Альтернатива? Одна строчка в `.zshrc`.

## Метод

```bash
alias cc='claude --dangerously-skip-permissions'
```

Всё. Одна строка. Теперь `cc` запускает Claude Code без единого вопроса.

Бash-команды? Выполняются. Файлы пишутся? Пишутся. Git push? Пушится. Package install? Ставится.

Без вопросов. Без пауз.

## Почему это работает

Авторизационные промпты — это **искусственные барьеры**. Они защищают от гипотетических угроз, но создают реальное трение каждый день.

Когда я впервые попробовал bypass-режим, было страшно. Помню第一次: "А если он удалит что-то важное?".

Результат: за месяц использования — ноль инцидентов. Почему?

**Потому что опасные действия — это не проблема Claude, это проблема твоего рабочего процесса.**

Claude делает то, что ты ему велишь. Если ты доверяешь себе довольно, чтобы запустить `rm -rf node_modules`, ты доверяешь себе достаточно, чтобы написать это в промпте.

##Guardrails, которые работают

Bypass-режим не значит бездумное выполнение. Я использую другие предохранители:

### 1. Global hooks как страховка

```bash
# ~/.claude/hooks.json
{
  "hooks": [
    {
      "id": "github-issues-push",
      "event": "PostToolUse",
      "tool": "Bash",
      "command": "bash ~/.claude/hooks/github-issues-push.sh"
    }
  ]
}
```

После каждого push скрипт проверяет: не забыл ли ты обновить issue? Не оставил ли что-то без коммита?

### 2. Правило двух ошибок

Claude ошибся дважды? `/clear` и свежий контекст. Это моё правило. Агент получает вторую попытку, но не третью.

### 3. Git как time machine

Всё в git. Каждый коммит. Любое изменение локального файла — коммит с осмысленным сообщением. Если что-то пойдёт не так — откат в одну команду.

### 4. Explicit NEVER rules в CLAUDE.md

```markdown
## Explicitly NEVER
- Никаких remote push без подтверждения
- Никаких изменений git config
- Никаких force push
```

Это работает лучше, чем любой approve prompt. Потому что это часть рабочего процесса, а не всплывающее окно которое можно "случайно" пропустить.

## Когда точно НЕ стоит использовать bypass

### 1. Секреты в env variables

Claude видит всё, что видит твоя shell. Все API keys, токены, пароли — в зоне доступа.

**Правило:** Используй только keys без финансового или sensitive-доступа.

### 2. Свежекломированные репозитории

В дикой природе встречаются `CLAUDE.md` с prompt injection и `package.json` с вредоносными post-install скриптами.

**Правило:** Если репо не твой — сначала review, потом работай.

### 3. Production серверы с публичным IP

У меня был случай: Clawdbot на VPS. Bypass mode + white IP + долгая сессия = через два дня утекли API keys.

**Правило:** Bypass + публичный IP = showcase, не production environment.

## Философия

Approve-mode как游泳 с нарукавниками. Ты не чувствуешь воду, но и не учишься плавать по-настоящему.

С bypass опасные действия vs безвредные становятся **видимыми через реальный feedback loop**, а не через абстрактное "разрешить/запретить".

Когда я случайно написал `claude "rm -rf /*"` — результат был виден сразу. Я увидел ошибку и исправил workflow, а не кликнул "разрешить" и удивлялся потом.

Это и есть суть: **обратная связь важнее защиты**.

## Результат после месяца

| Метрика | До | После |
|---------|-----|-------|
| approve prompts/day | ~88 | 0 |
| Time-to-task completion | размыто | стабильно |
| Trust in Claude | средний | высокий |
| Случайные ошибки | 0 | 0 |

Да, ноль случайных ошибок. Потому что когда нет искусственных барьеров — ты начинаешь реально думать о том, что запускаешь.

## TL;DR

```bash
# ~/.zshrc
alias cc='claude --dangerously-skip-permissions'
```

**Используй если:**
- 50+ approve prompts в день
- Работаешь в familiar repos
- Git как часть workflow

**Не используй если:**
- Секреты в env
- Публичный production server
- Новичок в Claude Code

В конце концов, это вопрос не безопасности Claude, а зрелости твоего рабочего процесса.
