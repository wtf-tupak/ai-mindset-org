# Plan Template — Orchestrator v2

> Этот файл создаётся оркестратором **динамически** при запуске задачи > 5 мин.
> Хранится в корне рабочей директории. Обновляется на каждом шаге.

---

## Формат

```markdown
# Plan: {название задачи}

## Meta
- **Created:** {ISO timestamp}
- **GitHub Issue:** #{номер}
- **Priority:** {low | normal | high | critical}
- **Assigned Agent:** {agent-name | orchestrator (self)}
- **Estimated Time:** {X min}

## Context
{Контекст задачи — откуда пришла, зачем, какие ограничения}

## Steps
- [ ] Step 1: {описание}
- [ ] Step 2: {описание}
- [ ] Step 3: {описание}
- [ ] Step 4: Оценить результат (quality_score)
- [ ] Step 5: Обновить GitHub Issue → закрыть

## Acceptance Criteria
- [ ] {Критерий 1}
- [ ] {Критерий 2}

## Notes
{Заметки, промежуточные результаты, lessons learned}
```

---

## Правила

1. **Создавай plan.md** только для задач > 5 мин или > 3 шагов
2. **Обновляй** чекбоксы `[ ]` → `[x]` по мере выполнения
3. **Не удаляй** завершённые шаги — они нужны для recovery
4. **Notes** — записывай промежуточные результаты и инсайты
5. **При компакции контекста** — plan.md сохраняется, status.md обновляется
