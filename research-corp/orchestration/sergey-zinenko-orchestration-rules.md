# Правила оркестрации моделей

## Принцип

Opus = мозг. Subagent'ы = руки. Opus думает и оценивает, subagent'ы делают. Экономия vs всё-на-Opus: ~60% токенов.

## Таблица делегирования

| Задача | Модель | Через |
|--------|--------|-------|
| Поиск файлов, grep, glob | **Haiku** | `Task(subagent_type="Explore", model="haiku")` |
| Запуск скриптов, проверка логов | **Haiku** | `Task(subagent_type="Bash", model="haiku")` |
| git status, railway logs, простые команды | **Haiku** | `Task(subagent_type="Bash", model="haiku")` |
| Написание/правка кода | **Sonnet** | `Task(subagent_type="general-purpose", model="sonnet")` |
| Написание текстов, постов | **Sonnet** | `Task(subagent_type="general-purpose", model="sonnet")` |
| Ресёрч, анализ данных | **Sonnet** | `Task(subagent_type="general-purpose", model="sonnet")` |
| Деплой на Railway, CI/CD | **Sonnet** | `Task(subagent_type="Bash", model="sonnet")` |
| Архитектура, проектирование систем | **Opus** | Сам, напрямую |
| Оценка качества subagent'ов | **Opus** | Сам, напрямую |
| Стратегия, планирование | **Opus** | Сам, напрямую |
| Голос автора (serg-voice, youtube-script) | **Opus** | Сам + скилл |

## Когда Opus делает сам (НЕ делегирует)

1. **Архитектура** — проектирование новой системы или крупный рефакторинг
2. **Оценка** — проверка результата subagent'а на качество
3. **Голос** — тексты, где важен стиль (посты, сценарии, чат от имени Сергея)
4. **Стратегия** — планирование, принятие решений, выбор подхода
5. **Мелочи < 30 сек** — если задача быстрее делегирования (поправить 1 строку, ответить на вопрос)

## Автоэскалация: когда subagent провалился

| Сигнал провала | Действие Opus |
|----------------|---------------|
| Haiku вернул "не нашёл" / пустой результат | Retry через Sonnet subagent с уточнённым запросом |
| Sonnet-код не работает после 1 попытки | Opus анализирует ошибку сам, фиксит или даёт Sonnet точные инструкции |
| Sonnet-текст generic / без голоса | Opus пишет сам с полным контекстом |
| Sonnet-ресёрч поверхностный | Opus subagent или Deep Research |
| Любой subagent вернул нерелевантный результат | Opus переформулирует задачу и retry на том же или высшем уровне |

## Правила сессии

- **Максимум 20-25 ходов** на сессию, потом `/compact` или новая сессия
- **Не читать файлы через Opus** — всегда через subagent (исключение: файл < 50 строк)
- **Параллельные subagent'ы** — если 2+ независимых задачи, запускать одновременно
- **Background subagent'ы** — для долгих задач (транскрипция, деплой) использовать `run_in_background: true`

## Антизависание

1. **Перед субагентом:** всегда писать `[Haiku/Sonnet] делаю X...`
2. **После субагента:** краткий отчёт что получил (1-2 строки)
3. **Мёртвая пауза запрещена:** максимум 1 ход без активного действия
4. **Background-задачи:** проверять через `TaskOutput(block=false)` каждые 60 сек
5. **Context switch:** при старте из summary/compact — ПЕРВЫМ делом доложить что было сделано и что дальше

## Индикация пользователю

- `[Haiku] ищу файлы...`
- `[Sonnet] пишу код...`
- `[Opus] оцениваю результат...`
- `[Escalation] Sonnet не справился, делаю сам`
