# Learning Loop

## Цель

Заполнять и обновлять раздел `Learning Loop` в любом целевом файле POS, чтобы переводить материал из состояния "не понял" в "понял" через итерации.

## Когда использовать

- После воркшопа или сессии разбора.
- После изучения статьи/документа/треда, если нужно зафиксировать понимание.
- В любой момент, когда нужно обновить статус тезисов и список задач по конкретному файлу.

## Где хранится результат

- В том же файле, к которому применяется цикл (`target file`).
- Отдельный файл `*-learning-loop.md` не создавать.

## Поддерживаемые цели (примеры)

- `workshops/YYYY-MM-DD-topic.md`
- `PhD/.../*.md`
- `threads/...md`
- любой markdown-файл внутри POS

## Формат раздела Learning Loop

1. `### Саммари`
2. `### Валидация тезисов`
   - `#### Понял`
   - `#### Не понял`
3. `### Задачи (по порядку выполнения)`
   - `#### Проект PhD`
   - `#### Воркшоп/скиллы`
4. `### Ссылки на артефакты`

## Алгоритм применения

1. Определи целевой файл (`target file`), где должен жить Learning Loop.
2. Открой файл и найди раздел `## Learning Loop`.
3. Если раздела нет — добавь его с полной структурой.
4. Если раздел уже есть — обнови пункты по фактам текущего состояния.
5. Переноси тезисы из `Не понял` в `Понял` только при наличии подтверждения (артефакт/результат).
6. Обновляй задачи в порядке выполнения, опциональные шаги помечай `(*)`.
7. Синхронизируй `Ссылки на артефакты` с фактически созданными/измененными файлами.

## Правила качества

1. Learning Loop обновляется в одном и том же целевом файле, где живет основной контекст.
2. Никаких дубликатов с отдельными `*-learning-loop.md`.
3. Каждый новый вывод должен быть проверяемым (факт, файл, результат).
4. Формулировки короткие, практичные, без лишней теории.
