# Драфт документа «Roi Развитие»

Еженедельная встреча по средам. Продукт — Roi Navigator. Документ к встрече: пункты для обсуждения (свободная форма) и в конце блок заметок, чтобы не забыть.

## Когда применять

- Пользователь просит драфт документа к встрече «Roi Развитие», к среде, к еженедельной встрече.
- Утро среды / подготовка к встрече.

## Формат документа

1. **Заголовок** — дата встречи и название: `DD.MM Roi Развитие` (например `11.03 Roi Развитие`).
2. **Пункты для обсуждения** — список или короткие блоки в свободной форме (что обсудить на встрече).
3. **В конце** — отдельный блок **«Заметки»** (или «Чтобы не забыть»): пустые пункты или подсказки, куда пользователь допишет своё.

## Шаблон драфта

```markdown
# DD.MM Roi Развитие

## Пункты для обсуждения

- 
- 
- 

## Заметки (чтобы не забыть)

- 
```

Дату подставлять на **ближайшую среду** (если не указана другая). Формат даты в заголовке: `DD.MM`.

## Улучшение драфта по образцам

Если в папке скилла есть файл **`past-meetings-sample.md`** с вставленным текстом прошлых встреч из Google Docs — использовать его, чтобы:
- повторить типичную структуру пунктов;
- предложить повторяющиеся темы (продукт, клиенты, процессы, команда и т.д.), если они видны в образцах.

Пользователь может один раз скопировать 1–2 документа с прошлых встреч в `past-meetings-sample.md`, чтобы следующие драфты были ближе к привычному формату.
